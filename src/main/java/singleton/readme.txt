单例模式（单件模式）
    适用于对某种有限资源的访问权限的控制。对该资源进行封装，使程序在任何时刻任何地点访问的都是当前存在的同一个管理对象。
    该模式保证一个类在任何时刻都仅有一个实例。
    该模式分为懒汉模式（需要使用实例时才生成，延迟加载）和饿汉模式（加载类的时候就生成实例，可能产生垃圾）。
模式角色：
    1、单件类（singleton）：创建出一个实例，并提供一个访问它的全局访问点。
使用场景：
    1、程序需要使用多线程且使用的时机无法确定，但是服务器的cpu数量固定。为最大限度的压榨cpu且尽可能减低锁竞争，
    需要对同一时刻的线程数量进行限制，程序需要使用线程池。需要保证程序各处获取的线程池对象都是同一个，才能保证
    对线程数量的控制。这就要求线程池对象是单例的。
    2、与线程池需求一样的还有数据库的链接池。
    3、对日志文件的内容追加。
