策略模式（政策）
    适用于只有一个出入口的程序，程序对于不同的参数有不同操作（有很多if else 语句）。
    要求客户端明确知道每个策略的作用。
    该模式利用了子类向上转型的特性，以父类做形参可以接受不同子类做实参。
模式角色：
    1、策略:具体策略的抽象接口，抽象出程序的功能要求具体策略实现并通过上下文角色进行调用。
    2、具体策略：策略接口的各种具体实现。
    3、上下文：供客户端使用的外观类，方法形参为策略接口类型，方法中调用接口方法。
使用场景：
    1、压缩类的实现
    要求实现一个压缩类，通过该类可以实现不同的压缩方式。当前以确定要求实现zip和gzip压缩方式。
    后续是否增加压缩方式未知，但程序设计应有扩展性，扩展时最大程度不修改现有代码。
    2、供客户端调用的支持不同支付方式的接口
    要求实现一个可供客户端选择不同支付方式的接口，客户端可以自由选择支付方式。后续可能增加其他
    类型的支付方式，所以程序要具有可扩展性，扩展时尽量不修改现有代码。
    3、通过jar包实现批量辅助程序
    要求设计一个jar包，实现shell脚本不方便实现的功能。jar包只有一个出入口，根据不同批量传递的参数
    实现不同的功能。设计时限制jar包只有一个出入口的main方法对外暴露，出入口接收外部程序的参数（实现功能类
    的完全限定名，用于反射生成具体策略类）。根据参数调用上下文环境的具体方法。
