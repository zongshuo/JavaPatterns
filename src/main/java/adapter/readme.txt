适配器模式（包装器）
    将功能相同或相似的一个接口的实现类包装成另一个接口的实现类，达到以a接口的实行使用b接口功能的目的。
双向适配器
    如果适配器角色同时实现了目标接口和被适配者接口，并包含二者的引用变量，那么这就是一个双向适配器。
单接口适配器
    如果适配器角色只需要适配一个接口，即适配器实现了一个接口并默认实现了接口的所有方法，使得其子类只需要重写任一方法就可以实现其功能，那么这就是一个单接口适配器。

    适配器模式可以在不修改已有类的情况下，使已有类适应新的需求。
    单接口适配器可以使子类不必实现接口的全部方法。
适配程度：
    1、完全适配：目标接口和被适配者拥有的方法数据相同，则适配器可以将二者的方法完全适配。
    2、不完全适配：目标接口的方法数量少于被适配者的方法数量，则适配器只能将二者部分适配。
    3、剩余适配：目标接口的方法数目比被适配者多，则适配器能将被适配者完全适配，但是必须给出目标接口剩余方法的默认实现。
优缺点：
    1、使目标和被适配者完全解耦
    2、满足开闭原则，放添加一个被适配者的新类时不用修改adapter。
接口角色：
    1、目标：是一个接口，该接口是客户想使用的接口
    2、被适配者：是一个已经存在的接口或抽象类，这个接口或抽象类需要适配。
    3、适配器：适配器是一个类，该类实现了目标接口并且包含被适配者的引用。
使用场景：
    1、当不愿意为了一个应用而修改各原接口的时候
    2、当没有对象的源代码的时候
