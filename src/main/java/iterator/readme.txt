迭代器模式（游标）
    提供一种方法顺序访问一个聚合对象中的各个元素，而又不需要暴露该对象的内部表示。
    一种可以让客户不必知道内部数据结构就可以遍历集合的模式。
    模式的关键是将遍历集合的任务交给一个称作迭代器的对象。
优缺点：
    1、用户使用迭代器遍历集合时，不需要了解对象在集合中是如何表示及存储的。
    2、用户可以同时使用多个迭代器遍历同一个集合。
模式角色：
    1、集合：一个接口，规定了具体集合需要实现的操作。
    2、具体集合：具体集合是实现集合接口的类的实例，具体集合按照一定结构存储对象。具体集合应该有一个方法用于返回一个针对该集合的具体迭代器。
    3、迭代器：一个接口，规定了遍历具体集合的方法。如next();
    4、具体迭代器：实现迭代器接口的类的实例。具体迭代器在实现迭代器接口所规定的遍历集合的方法时，
    要保证首次调用将按着集合的数据结构找到该集合中的一个对象，而且每当找到集合中的一个对象，立刻
    根据该集合的存储结构得到待遍历的后继对象的引用，并保证依次调用遍历方法可以遍历集合。
使用场景：
    1、让用户访问一个集合中的对象，但不想暴露对象在集合中的存储结构。
    2、希望对遍历不同的集合提供一个统一的接口。
    3、Java中所有集合都实现的迭代器。
